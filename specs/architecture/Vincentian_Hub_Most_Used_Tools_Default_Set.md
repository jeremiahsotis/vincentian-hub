# Vincentian Hub – Recommended Default “Most Used Tools” Set

This is the recommended default set for the **Most Used Tools** section for standard Conference-side Vincentian users.

These are ordered for real use frequency, not abstract importance.

## Recommended default set (Conference users)

### 1. Resource Database
**Why it belongs here:**  
This is one of the highest-frequency practical actions a Vincentian will take.

**Behavior:**  
Auto-inject the user’s Conference context so available resources are already filtered correctly.

**Label shown to users:**  
Resource Database

### 2. Conference Map
**Why it belongs here:**  
Users need to confirm service areas quickly and avoid confusion about which Conference serves which address.

**Label shown to users:**  
Conference Map

### 3. Clothing Voucher
**Why it belongs here:**  
This is a direct action tool and should be one tap away.

**Behavior:**  
Auto-inject the user’s Conference context.

**Label shown to users:**  
Clothing Voucher

### 4. ShyftUnity / ServWare
**Why it belongs here:**  
Case tools should be immediately accessible for users who rely on them regularly.

**Label shown to users:**  
Neighbor Records  
or  
ShyftUnity / ServWare

**Recommendation:**  
Use the plain-language label **Neighbor Records** with smaller helper text naming the system.

### 5. Community Harvest Vouchers
**Why it belongs here:**  
If this is used regularly enough, it belongs in Most Used Tools rather than buried in documents.

**Label shown to users:**  
Community Harvest Vouchers

### 6. Upcoming Events
**Why it belongs here:**  
This is not a “tool” in the strict sense, but users often need to know what is coming up next. It should be visible near the top of the dashboard, though it can live as its own panel rather than inside the tile grid.

**Label shown to users:**  
Upcoming Events

## Items that should NOT be in the default Most Used Tools grid

These should still exist in the dashboard, but not in the primary top tool block unless a specific Conference has a strong reason.

### Real Life Finance
Better placed in:
- Learning / Training
- Member Resources
- Tools & Learning

### Understanding Evictions in Indiana
Better placed in:
- Learning / Training
- Documents / Guides
- Legal / Housing Guidance

### General Google Drive-derived document libraries
Better placed in:
- Forms & Documents
- Leadership Resources
- Reflection & Support

### Special Purpose Grants
Better placed in:
- Announcements / What’s New
- Grants
- Featured opportunities

Reason: grants are often temporary and should not displace the most stable high-frequency tools.

## Recommended default visual layout

For mobile:
- 4 primary tool cards in a 1-column or 2-column responsive grid
- Community Harvest as either the 5th card or first item below the grid
- Upcoming Events immediately below Most Used Tools

## Strongest recommended top 4
If you need the cleanest possible MVP default, use these four first:

1. Resource Database
2. Conference Map
3. Clothing Voucher
4. Neighbor Records

That set is simple, high-frequency, and easy for aging users to understand.

## Plain-language labeling recommendation

Prefer labels that describe the task instead of the system name.

Examples:

- **Neighbor Records** instead of only “ServWare”
- **Clothing Voucher** instead of “Virtual Clothing Voucher”
- **Find Resources** as an alternate label for “Resource Database”

Do not over-rename things if users already know the existing labels, but where possible the visible label should feel direct and human.

## District user variation

For Assistance Line / district operational users, the default Most Used Tools set should be different:

1. Resource Database
2. Clothing Voucher
3. Conference Map
4. Neighbor Records
5. SOPs / Procedures
6. Pricing / Reference Docs

That dashboard should be operational, not leadership-oriented.

## Final recommendation

For standard Conference users, lock the default Most Used Tools set to:

- Resource Database
- Conference Map
- Clothing Voucher
- Neighbor Records
- Community Harvest Vouchers

And show **Upcoming Events** immediately below that block rather than inside it.
